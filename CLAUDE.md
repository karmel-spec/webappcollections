# The Larson Gallery — webappgallery.com

## What this is
A showcase site for all of Karmel's webapps, styled as a gallery wall —
framed screenshots, per-app detail pages, build phases, a queue of what
gets painted next, and Netlify feedback forms. Live at webappgallery.com
(Netlify; every push to `main` auto-deploys production).

## Golden rule: template.html is the source
1. NEVER edit `index.html` directly — it is generated output.
2. Edit `template.html`, then rebuild:
   `python3 build.py`
   This regenerates `index.html` (screenshots + icons embedded as data
   URIs, ~2 MB) AND the per-app share pages under `a/<id>/index.html`.
3. Commit `template.html`, `index.html`, and `a/` together after a build.
4. `build.py` reads `GALLERY_BASE_URL` (default `https://webappgallery.com`)
   for absolute OG image URLs on share pages.

## Structure
- `template.html` — everything: CSS, HTML sections, and the JS app data.
  - `const APPS = [...]` (~line 565): one object per app —
    `{id, name, cat, frame, big?, archived?, tag, phase, pct, stack,
    born, github, live, story}`.
  - `const PHASES`: dreaming → sketching → painting → retouching →
    hung ("Ready") → archived. Each has a CSS color var (`--ph-*`).
  - The Queue section (~line 422): hand-edited ordered list of upcoming
    apps; keep the count in the note below it in sync.
  - `__SCREENS_JSON__` / `__ICONS_JSON__` placeholders — build.py fails
    if either is missing.
- `assets/screens/` — JPGs named `<id>-main.jpg`, `<id>-d1..d4.jpg`
  (desktop), `<id>-m1..m2.jpg` (mobile). `-main` is required per app
  (it's the share-page OG image); d/m shots are optional and picked up
  automatically.
- `assets/icons/` — `<id>.png` home-screen icons.
- `a/<id>/index.html` — generated share pages (OG tags + redirect into
  `index.html#/app/<id>`). Regenerated by build.py; don't hand-edit.
- `netlify.toml` — static publish of `.`, no build step on Netlify;
  building happens locally before commit.

## Adding a new app
1. Add screenshots to `assets/screens/` and an icon to `assets/icons/`
   using the id-based naming above. Blur any private data in screenshots
   before adding — this site is public.
2. Add the app object to `APPS` in template.html (match the existing
   field order — build.py regex-parses `id/name/cat/tag` from it).
3. If it was in The Queue, remove it there and fix the queue count.
4. `python3 build.py`, then commit everything including the new `a/<id>/`.

## Forms
Netlify Forms: `app-request` (the Queue request form) and `app-feedback`
(per-app, rendered dynamically). Three moving parts — keep all of them:
- `forms.html` — tiny static page whose only job is form REGISTRATION with
  Netlify's detector (index.html is ~2 MB and may be skipped by
  post-processing). Its field names must match what the site posts.
- Both forms submit via `ajaxify()` in template.html (fetch POST to `/`,
  on-brand toast, stays in the SPA); native fallback posts to `/thanks/`
  (styled static page).
- Netlify requires **form detection enabled** in Site configuration →
  Forms — without it every POST returns 404. If forms 404, check that
  toggle first, then redeploy.
Submissions land in the Netlify dashboard (set up email notifications
there too).

## Voice & design
Gallery-wall metaphor throughout: apps are paintings, phases are studio
stages ("painting," "retouching," "hung"), sections are walls. Warm
editorial ivory palette (`#EDE3D3` paper, `#3D3428` ink, `#A88448` brass),
Charter/Georgia serif. Copy is first-person Karmel — warm, modest,
story-first, lowercase-friendly taglines. Keep it refined and editorial,
never flyer-like.

Momni-related copy on this site must still follow Momni's language rules:
never "vetted/verified/screened/safe/guaranteed," and never name Momni's
former lead investor anywhere.
